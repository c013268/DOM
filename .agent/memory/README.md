# Agent Memory — Overview

This folder stores accumulated project knowledge that agents reference across sessions.

## Structure

| File | Purpose |
|------|---------|
| `implementations.md` | New code implementations, features added, and design decisions made |
| `gotchas.md` | Edge cases, tricky behaviors, and non-obvious patterns discovered |
| `errors-and-fixes.md` | Errors encountered during development and their resolutions |
| `business-logic.md` | User-defined business rules, data transformations, and domain logic |
| `conventions.md` | Project-specific conventions, patterns, and standards discovered |

## Usage
- Agents should consult these files before making changes
- After discovering new information, agents should append to the relevant file
- Entries should be dated and concise
- Each entry should be actionable (not just observational)
